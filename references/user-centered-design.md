# 以用户为中心的设计（UCD）

应用 ISO 9241-210 UCD 框架，创建设计优化以适配特定用户。

## 何时使用

- 设计新界面或系统时
- 根据用户需求重新设计现有产品时
- 创建游戏界面或交互系统时
- 建立以用户为中心的设计流程时

## 核心 ISO 六要素

1. **全程用户参与** — 在设计过程的每个阶段都要让用户参与
2. **以用户为中心评估** — 让用户测试驱动迭代设计决策
3. **完整用户体验** — 关注完整体验，而非孤立功能
4. **多学科视角** — 整合各领域洞察

## 需要回答的关键问题

- **用户是谁？** 定义年龄、性别、经验水平、背景
- **他们的目标和任务是什么？** 理解他们想要完成什么
- **他们的环境是什么？** 考虑场景（手机 vs PC，嘈杂 vs 安静，公共 vs 私人）
- **他们有什么限制？** 考虑学习曲线、身体限制、无障碍需求

## 流程工作流

1. **创建用户画像**
   - 开发代表真实用户类型的虚构身份
   - 包含人口统计、行为、动机和目标

2. **发展场景**
   - 描述用户与系统交互的背景和环境
   - 建立使用情境、条件和情况

3. **定义用例**
   - 分解可能的用户行为和预期结果
   - 逐步映射交互流程

4. **用真实用户验证**
   - 测试行为和学习曲线的预测
   - 根据实际用户反馈迭代设计

## 约束条件

- UCD 可能耗时且昂贵
- 需要接触目标用户进行研究与测试
- 需要预先明确界定目标受众
