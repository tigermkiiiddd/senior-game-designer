# 详细设计方法论参考

## 头脑风暴：高级技术

### 组合方法

对于复杂项目，组合多种头脑风暴方法：

1. **从词语气泡开始**进行初步概念探索
2. **过渡到树状图**进行功能组织
3. **使用流程图**进行机制流程
4. **应用身体风暴**进行控制方案验证

### 团队头脑风暴

当与团队头脑风暴时：
- 指派一人记录所有想法
- 想法生成阶段不要批评
- 明确地建立在他人想法上
- 设置时间限制以保持精力
- 之后审查和分类

## 核心循环：深度分析

### 循环持续时间考量

| 持续时间 | 类型 | 示例 |
|----------|------|----------|
| 1-5 秒 | 微循环 | 开枪、跳跃 |
| 5-30 秒 | 动作循环 | 战斗遭遇、谜题 |
| 30 秒 - 5 分钟 | 会话循环 | 关卡完成、比赛 |
| 5-30 分钟 | 游戏会话 | 任务完成 |
| 小时 | 元循环 | 角色进阶、赛季 |

### 循环失败模式

**模式 1：无深度**
- 循环有趣一次但不进化
- 玩家快速掌握并无聊
- 解决方案：添加复杂性层次，与其他机制组合

**模式 2：无反馈**
- 玩家不理解动作的影响
- 循环感觉无意义
- 解决方案：添加清晰视觉/音频反馈，进展指示

**模式 3：无灵活性**
- 循环只在一种情况下有效
- 无法扩展到新场景
- 解决方案：设计模块化动作，以多种方式组合

**模式 4：无扩展**
- 无法向循环添加新动作
- 游戏无法成长
- 解决方案：为未来机制设计钩子

### 小游戏应用

并非所有游戏都需要光环级别的野心：

- **糖果传奇**：简单的三消循环，具有显著重复性
- 小游戏可以验证大型体验的核心循环
- 用作原型工具（见原型制作技能）

## 迭代：高级模式

### 迭代螺旋

```
        版本 4
            ↑
    版本 3
        ↑
版本 2
    ↑
版本 1（原型）
```

每次迭代应该：
1. 添加有意义的内容
2. 改进现有机制
3. 修复已识别的问题
4. 走向最终愿景

### 迭代范围管理

**每次迭代：**
- 专注于 1-3 个主要添加
- 修复上一次迭代的关键 Bug
- 在下一次迭代前收集反馈

**避免：**
- 每次迭代添加太多功能
- 开发中途改变核心机制
- 迭代之间跳过测试

### 回归测试

每次迭代后：
- 测试所有之前工作的功能
- 新功能可能破坏旧功能
- 为每次迭代维护测试清单

### 不同游戏类型中的迭代

**线性游戏（银河战士风格）：**
- 玩家能力迭代（成长）
- 用新能力返回之前区域
- 每个区域为多种能力水平设计

**街机游戏（大金刚风格）：**
- 用增加难度循环整个游戏
- 相同内容，不同参数
- 敌人速度、生成率、AI 侵略性

**服务游戏（MMO、实时服务）：**
- 内容更新作为迭代
- 季节变化、新区域、平衡补丁
- 元游戏在多年中演进
