# 游戏设计方法论

## 何时使用

- 头脑风暴游戏概念或机制
- 设计或评估核心玩法循环
- 规划迭代开发周期
- 根据测试完善游戏机制
- 创建进阶系统

## 设计工作流

### Phase 1：头脑风暴

根据开发阶段和结构需求选择方法。

#### 自由联想方法（早期开发）

**词语气泡与概念网：**
1. 将一个或多个词语/想法作为起点写下
2. 添加更多词语/想法并连接到起点
3. 向外扩展，连接到起点和其他想法
4. 结果：类似蜘蛛网结构的想法图谱

**最适合：** 设计师只有基本游戏想法或情节概念

#### 结构化思维方法（后期开发）

**流程图：**
1. 从词语气泡的起点开始
2. 构建连接到起点的结构
3. 每个想法延伸到下一步，无分支

**最适合：** 设计有序结构、顺序机制

**树状图：**
1. 与流程图相同的起始方法
2. 将想法组织成分层、层级结构

**最适合：** 创建层级项目或动作、技能树、进阶系统

#### 身体表达方法

**身体风暴（Bodystorming）：**
1. 用身体动作辅助创意过程
2. 表演角色动作或控制方案
3. 对动作控制技术特别有帮助

**核心原则：** 头脑风暴不应局限于写作——要身体力行地演示

### Phase 2：核心游戏循环设计

核心游戏循环是玩家长期满意度的**关键**。

#### 定义

核心游戏循环 = 一种玩家觉得有趣且愿意持续重复的玩家驱动行为。

#### 循环结构

```
┌─────────────────────────────────────┐
│                                     │
│    玩家行动                          │
│           ↓                         │
│    行动结果                          │
│           ↓                         │
│    玩家反应                          │
│           ↓                         │
│    游戏要求重复                       │
│           ↓                         │
│    玩家行动（循环）                   │
│                                     │
└─────────────────────────────────────┘
```

#### 设计要求

核心循环动作必须满足：

| 要求 | 描述 |
|------|------|
| 易理解 | 目的和结果清晰 |
| 易操作 | 输入/响应直观 |
| 有趣 | 单独玩也好玩 |
| 直接反馈 | 玩家看到即时结果 |
| 灵活性 | 适应不同场景 |
| 可扩展 | 可添加额外动作 |
| 可组合 | 与其他动作配合 |
| 可进化 | 支持其他游戏循环 |

#### 组成

- 通常由一系列**动词**组成（越具体越好）
- 示例：驾驶、射击、潜行
- 应该用几句话就能描述
- 应该清楚解释体验中最吸引人的部分

#### 示例

**超级马里奥兄弟：**
- 核心循环：跳跃
- 整个游戏围绕跳跃动作构建
- 其他动作插入：杀敌、顶砖、切换场景
- 保持新鲜感：各种挑战、不同的动作组合

**光环系列——"30秒的乐趣"：**
- 延伸传统 5-10 秒的核心循环
- 循环：武器序列 → 扔手雷 → 近战 → 短暂恢复 → 下一场战斗
- 保持冲击力：新武器、敌人、地形、外部因素

#### 关键警告

如果核心循环有缺陷 → 几乎不可能通过整合其他元素来创造有趣体验。

没有深度和灵活性的核心循环不会让玩家长时间留存。

### Phase 3：迭代开发

迭代 = 重复执行一个动作，用前一次结果作为下一次的基础。

#### 开发过程

1. **从原型开始**：粗糙图形（形状和字母），最少功能
2. **每次迭代增加：**
   - 更多功能点
   - 更多游戏功能
   - 现有机制变得更复杂
   - 图形和声音更好地呈现愿景
3. **结果**：更精致、响应更灵敏、更易上手的游戏

#### 迭代类型

**内部迭代（开发中）：**
- 每个版本都比前一个改进
- 从"骨架"到"有血有肉"的渐进演化

**外部迭代（玩家可见）：**
- 年货体育游戏更新
- 续作和扩展包
- DLC 和内容更新

**游戏内迭代：**
- 塔防：每一波都是更难的一轮
- 银河战士：玩家能力成长，可以回到之前区域
- 大金刚：循环回到第一关但难度增加

#### 难度即迭代

- 每个难度等级都建立在前一个难度的基础之上
- 玩家遇到基于之前技能的新挑战
- 更高难度 = 在核心机制上的迭代加上增加的挑战

## 整合检查清单

- [ ] 根据开发阶段选择头脑风暴方法
- [ ] 用具体动作动词定义核心循环
- [ ] 核心循环满足所有设计要求
- [ ] 建立迭代计划用于开发
- [ ] 识别保持循环新鲜感的变数因素
