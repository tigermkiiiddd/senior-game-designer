# 视觉玩家引导

设计视觉元素，直观地传达互动可能性，并在不需要教程或明确指示的情况下引导玩家注意力。

## 何时应用

- 设计交互式 UI/HUD 元素
- 创建需要玩家导航的游戏环境
- 设计带有物理或虚拟物体的谜题
- 实现触屏/平板手势系统
- 构建需要路径指引支持的关卡

## 核心原则

### 可供性暗示

可供性暗示是直观地传达如何与物体互动的视觉或物理特征。

**关键规则：** 暗示必须匹配设计师预期的动作。打算被按下的按钮必须看起来可按，而不是可拉。

#### 应用领域

**UI/HUD 设计：**
- 使用倒角、阴影和形状（如圆形）来暗示可点击性
- 设计吸引感官的元素（美味的、触觉的）以增加互动冲动
- 确保形式追随功能——旋钮应暗示圆形手势

**触屏/平板界面：**
- 将自然动作（滑动、点击）映射到视觉暗示
- 使用引导手指移动的视觉指标（例如圆形追踪路径）

**环境和谜题：**
- 使用清晰路径（道路、地毯）引导玩家移动
- 设计具有手柄或暗示正确行为的特征的物体
- 从基本形状开始，然后添加暗示（颜色、阴影）以明确互动

### 注意力捕获

使用先天的心理触发器将玩家注意力集中在特定元素上。

**用于营销/UI：**
- **面孔**：人类本能地识别和关注面孔
- **运动**：闪烁、弹跳或动画元素吸引视线
- **意外**：在熟悉界面中打破模式以创造兴趣
- **吸引人的图像**：食物、吸引人的角色触发基本欲望

**用于寻路（关卡设计）：**
- **地标**：决策节点处的独特物体用于定向
- **引诱物**：吸引玩家的大型远处地标
- **产道**：引导到开阔空间的漏斗形空间（张力/释放）
- **结构化路径**：引导移动的视觉线条（道路、清理的森林）
- **光线**：玩家本能地向光移动；暖光 = 舒适，冷光 = 令人生畏

## 执行工作流

1. **为每个交互元素识别预期的玩家动作**
2. **选择匹配预期动作的适当视觉暗示**（按、拉、旋转等）
3. **测试直觉性**：玩家能否在无指示的情况下理解互动？
4. **应用注意力捕获技术**突出重要元素
5. **为环境中的导航实施寻路暗示**
6. **验证一致性**：确保整个游戏中暗示匹配

## 变量

- **visual_cue**：表示用法的形状、阴影、倒角或颜色
- **intended_action**：设计师希望用户执行的特定行为
- **cue_type**：类别（面孔、运动、地标、光线等）
- **goal**：注意力捕获或导航

## 好处

- 减少花在教程/指示上的时间
- 直观地引导玩家采取正确行为
- 防止游戏玩法变成"逻辑灾难"
- 在无明确指示的情况下引导玩家注意力
