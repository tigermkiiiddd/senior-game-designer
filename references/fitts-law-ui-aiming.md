# Fitts 定律用于 UI 和瞄准机制

## 何时使用
- 设计带有可点击/可点击元素的用户界面
- 创建战斗或互动瞄准机制
- 优化控制器或鼠标输入系统
- 平衡基于精度的游戏难度
- 减少用户疲劳和沮丧

## 核心概念
移动时间取决于到目标的距离和目标的大小。随着速度增加，精度降低——这是一个基本的权衡。

**关键参数：**
- D = 从起点到目标中心的距离
- W = 沿运动轴的目标宽度
- H = 目标高度（对可用性至关重要）

## 执行步骤

### 1. 分析移动任务
- 确定用户从哪里开始（光标位置、角色位置）
- 确定目标（按钮、敌人、交互元素）
- 测量距离（D）和目标大小（W，H）

### 2. 应用 UI 设计原则

**将相关元素分组**
- 将相似的功能元素放在一起
- 减少相关动作之间的移动距离
- 创造逻辑互动模式

**优化元素放置**
- 将常用元素放在常见起始位置附近
- 示例：将主要动作放在光标通常停留的位置附近
- 考虑屏幕边缘和角落（实际上是无限大的目标）

**适当调整元素大小**
- 将常用元素做得更大
- 增加宽度（W）以减少所需精度
- 确保足够的高度（H）——细目标难以点击

**减少距离**
- 将 UI 元素移近互动点
- 最小化不必要的游标移动
- 考虑径向菜单以加快访问

### 3. 应用于战斗/瞄准机制

**设置难度参数**
- 根据期望难度调整目标大小（W）
- 调整距离（D）以挑战玩家精度
- 平衡两者以创造预期体验

**对照熟练玩家测试**
- 最快/最熟练的玩家设定难度上限
- 如果专家都打不中，就太难了
- 如果新手都打不中，考虑难度缩放

**考虑移动约束**
- 玩家移动速度影响有效距离
- 武器特性影响精度
- 环境因素（掩体、障碍物）修改 D 和 W

### 4. 通过测试验证
- 测量实际移动时间
- 观察用户沮丧和疲劳
- 用不同输入设备测试（鼠标、控制器、触摸）
- 根据真实用户表现迭代

## 常见应用
- **按钮放置：** 根据使用频率调整大小和位置
- **菜单：** 减少顺序选择之间的距离
- **战斗目标：** 敌人命中框和交战距离
- **快速反应事件：** 平衡时间窗口与目标区域
- **拖放：** 优化放置区域和移动路径

## 关键考量
- 适用人类身体极限
- 极小或极远的目标可能不可能
- 不同输入设备有不同的精度特性
- 疲劳随时间增加——考虑延长会话
- 可访问性：一些用户行动能力降低
- 视觉反馈可以帮助补偿小目标

## 难度平衡
- **简单：** 大目标、短距离
- **中等：** 中等目标和距离
- **困难：** 小目标、长距离
- **专家：** 非常小的目标、非常长的距离或移动目标
