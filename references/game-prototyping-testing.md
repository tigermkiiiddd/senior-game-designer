# 游戏原型制作与测试

## 何时使用

- 在全面开发前验证游戏概念
- 发现 Bug 和可用性问题
- 测试玩家体验和乐趣因素
- 确保游戏清晰传达
- 验证网络游戏服务器稳定性

## 原型制作方法

### 纸面原型

**最适合：** UI 设计、卡牌游戏、桌游、早期概念验证

**材料：** 纸、剪刀、笔

**优点：**
- 最快最便宜的方法
- 即时迭代
- 易于修改

**技巧：**
- 使用颜色/纹理区分元素
- 快速创建多个版本
- 立即用真实玩家测试

### 数字原型

**最适合：** 视频游戏机制、控制方案、时间敏感的游戏玩法

**工具：** 触屏、中间件、游戏引擎

**优点：**
- 初始设置后迭代更快
- 易于与远程团队共享
- 更准确地代表最终体验

### 一般原型制作原则

**目的：** 创建代表性模型来测试：
- 可行性（我们能建造这个吗？）
- 可用性（玩家能理解这个吗？）
- 市场需求（人们会想要这个吗？）

**警告：** 不要为了在早期节省时间而跳过原型制作。

## 测试方法

### 一次性测试（Kleenex 测试）

**定义：** 每个测试对象仅使用一次。

**目的：** 验证第一印象和初始学习曲线。

**流程：**
1. 带来从未见过游戏的全新玩家
2. 观察他们的第一次互动
3. 记录他们挣扎或困惑的地方
4. 不要对这个测试类型重复使用同一玩家

**最适合：** 教程设计、首次用户体验、入职

### 黑盒测试

**定义：** 对象对设计一无所知。

**目的：** 揭示自然玩家行为与设计师意图的对比。

**流程：**
1. 给玩家游戏，只提供基本控制说明
2. 观察他们自然地如何接触游戏
3. 注意他们的假设与设计意图的差异
4. 识别误导玩家的 UI/UX 问题

**最适合：** 发现非预期游戏玩法、找出混乱元素

### 白盒测试

**定义：** 对象遵循脚本或特定路径。

**目的：** 找出特定 Bug 和逻辑错误。

**流程：**
1. 创建覆盖特定功能的测试脚本
2. 引导玩家完成预定动作
3. 记录任何 Bug 或意外行为
4. 修复后，执行回归测试

**回归测试：** 在 Bug 修复后重新测试所有之前工作的功能。

**最适合：** QA、Bug 寻找、功能验证

### 负载测试

**定义：** 同时模拟数千玩家。

**目的：** 检查服务器稳定性和性能。

**应用：** MMO、在线多人游戏、实时服务游戏

**流程：**
1. 模拟预期玩家负载
2. 监控服务器性能
3. 识别瓶颈和故障点
4. 测试超出预期负载以获得安全边际

**最适合：** 上线前的网络游戏、重大更新

## 测试目标

确保游戏是：

| 目标 | 验证方法 |
|------|------------------|
| 有趣 | 一次性测试，观察情感反应 |
| 清晰 | 黑盒测试，追踪困惑点 |
| 功能正常 | 白盒测试，Bug 报告 |
| 稳定 | 负载测试，性能指标 |

## 测试循环

```
┌─────────────────────────────────────┐
│                                     │
│    1. 原型                          │
│           ↓                         │
│    2. 测试                          │
│           ↓                         │
│    3. 分析结果                       │
│           ↓                         │
│    4. 迭代                          │
│           ↓                         │
│    返回步骤 2                        │
│                                     │
└─────────────────────────────────────┘
```

## 快速参考

| 方法 | 何时使用 | 揭示什么 |
|--------|-------------|-----------------|
| 纸面原型 | 早期概念，UI/桌游 | 核心机制可行性 |
| 数字原型 | 视频游戏机制 | 感觉、时机、控制 |
| 一次性测试 | 首次体验 | 入职、教程 |
| 黑盒测试 | 自然行为 | UX 问题、假设 |
| 白盒测试 | Bug 寻找 | 技术问题 |
| 负载测试 | 网络游戏 | 服务器稳定性 |
