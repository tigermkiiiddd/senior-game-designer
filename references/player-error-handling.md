# 玩家错误处理

## 何时使用

- 在内测期间分析玩家行为
- 设计容错游戏系统
- 为玩家错误创建反馈
- 改善用户体验和可访问性
- 调试玩家流程问题

## 错误分类系统

### Level 1：主要类别

```
┌─────────────────────────────────────┐
│        玩家错误                      │
└─────────────┬───────────────────────┘
              │
    ┌─────────┴─────────┐
    ▼                   ▼
运动控制            行为
错误                 错误
```

### 运动控制错误

**定义：**源于玩家协调、掌握或应用输入设备的错误。

**示例：**
- 简单：意外按错按钮
- 复杂：未能正确把握 Boss 战动作时机
- 物理：意外打翻棋子

**设计回应：**
- 了解目标受众能力
- 进行内测以预测和控制这些错误
- 考虑玩家年龄或身体状况限制
- 示例：儿童桌游不应使用微小部件
- 主机游戏必须考虑手柄熟悉度

### Level 2：行为错误子类型

```
┌─────────────────────────────────────┐
│        行为错误                      │
└─────────────┬───────────────────────┘
              │
   ┌──────────┼──────────┐
   ▼          ▼          ▼
 流程       遗漏       错误
 错误       错误       行动
```

#### 流程错误

**定义：** 在流程中添加额外的、不需要的步骤。

**示例：** 玩家认为需要返回主菜单才能找到地图，而当前屏幕可以访问。

**原因：**
- 混乱的界面设计
- 无意视盲

**设计回应：** 改善 UI 清晰度，减少不必要的导航路径

#### 遗漏错误

**定义：** 在动作序列中未能完成一步，遗漏重要内容。

**设计回应：**
- 保持动作序列简短直接
- 确保步骤信息明显、逻辑或直观
- 适当平衡学习曲线

#### 错误行动错误

**定义：** 在情况下执行了错误动作——应该做一件事但做了另一件。

**设计回应：** 应用"不受惩罚的错误"原则

**提供：**
- 有趣的失败状态
- 机智、聪明的提示消息
- 无重大惩罚的恢复机会

## 设计响应框架

### 一般原则

1. **预测** 游戏可能的错误类型
2. **准备** 提前对策
3. **提供** 有帮助的提示文字
4. **引导** 玩家回到正确方向
5. **让** 偶尔的错误有趣而非沮丧

### 错误严重程度响应矩阵

| 严重程度 | 响应 |
|----------|----------|
| 轻微 | 微妙的视觉/音频暗示 |
| 中等 | 清晰反馈，轻松恢复 |
| 重大 | 明确指导，最小惩罚 |
| 严重 | 自动保存、检查点或撤销选项 |

### 目标受众考量

| 受众 | 运动控制重点 | 行为重点 |
|----------|---------------------|------------------|
| 儿童 | 大 UI 元素、简单输入 | 短序列、明显步骤 |
| 休闲 | 宽容时机、清晰提示 | 引导教程、提示 |
| 硬核 | 接受精确挑战 | 允许复杂序列 |
| 无障碍 | 替代输入方法 | 延长时限 |

## 快速参考

| 错误类型 | 关键特征 | 主要设计响应 |
|------------|-------------------|------------------------|
| 运动控制 | 输入协调 | 了解受众、内测 |
| 流程 | 多余的不必要步骤 | 清晰 UI，减少路径 |
| 遗漏 | 缺少步骤 | 短序列，明显信息 |
| 错误行动 | 错误选择 | 有趣的失败，恢复 |
