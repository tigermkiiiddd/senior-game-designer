# 游戏设计原理参考

## 何时使用

- 导航《游戏设计的100个原理》文档结构
- 按 ID 编号（1-100）定位特定原理
- 按主题类别（创新、创作、平衡、问题解决）寻求设计知识
- 在游戏开发中寻找灵感或解决方案
- 理解设计原理之间的关系

## 文档结构

100 个原理组织成**4 个主题部分**：

### 第一部分：游戏创新的一般原理（原理 1-29）
核心理论：玩家心理学、博弈论和基本设计概念。

### 第二部分：游戏创作的一般原理（原理 30-61）
实践聚焦：开发流程、原型制作和生产方法。

### 第三部分：游戏平衡的一般原理（原理 62-79）
系统设计：玩家参与、学习曲线和奖励结构。

### 第四部分：解决问题的一般原理（原理 80-100）
可用性和 UX：认知心理学、界面设计和玩家反馈。

## 使用策略

### 基于需求选择模式

1. **随机灵感**：当头脑风暴停滞时翻到随机原理
2. **概念复习**：参考特定原理以回忆复杂概念
3. **学习**：阅读不熟悉的章节以获得新见解
4. **问题诊断**：使用原理之间的相互联系识别根本原因
5. **问题解决**：查阅附录中结构化问题解决方法

### 导航惯例

- **引号中的橙色文字**：交叉参考另一个原理（当作超链接处理）
- **蓝色文字**：与概念关联的设计师姓名
- **布局**：左页文字解释，右页图解说明

### 范围考量

- 原理按组织分类，但适用于所有开发阶段
- 每个原理是 2 页介绍，非详尽处理
- 使用术语和关键词进行更深入的网络搜索
